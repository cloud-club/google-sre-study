# Chapter 5 - Eliminating Toil

## Summary
- 평범함의 의미는 성장에 따라 얼마든지 달라질 수 있다.
### 삽질
- 프로덕션 서비스를 운영하는 것과 직접적으로 연관이 있지만 번거롭고 자동화가 가능한 것
- 삽질은 서비스가 성장하면서 계속 증가한다.
- 삽질을 줄일수록 개발에 더 집중할 수 있다.
### 엔지니어링
- 새로우면서도 본질적으로 사람의 판단을 필요로 하는 업무
- 전략에 따라 서비스의 지속적인 개선을 이루며 창의적으로 문제를 해결
### SRE의 업무
1. 소프트웨어 엔지니어링
   1. 코드를 작성하거나 관련된 문서화 작업을 수행
2. 시스템 엔지니어링
   1. 한 번의 노력으로 지속적인 개선을 이루기 위해 프로덕션 시스템을 조정하거나 문서화를 수행
   2. 모니터링, 로드밸런싱, 아키텍처 조언 등
3. 삽질
   1. 서비스 운영과 관련된, 반복적인 수작업
4. 부하
   1. 서비스 운영과 직접 관련되지 않은 관리 업무

### 삽질의 장점
- 예측 가능하고 반복되는 작업은 성취감을 준다.
- 다만 빠른 시간 내에 처리할 수 있는 양까지만 괜찮다.
### 삽질의 단점
- 경력 개발이 침체된다.
- 의욕 저하
- 조직에게는 역할의 혼란이 발생할 수 있으며 좋지 않은 선례를 남긴다.
  - 나중에도 그만큼의 업무를 바라게 된다.
- 인력의 유출 발생 가능
  - 보람을 찾아 엔지니어가 떠날 수도...
- 신뢰 문제
  - 취업 사기..?
### 결론
- 삽질을 줄이고 창의적인 일에 더 집중하자
